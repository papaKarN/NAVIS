# NAVIS

**NA**nopore **V**isualization & **I**nteractive **S**tatistics

NAVIS is a quality control (QC) tool for Oxford Nanopore sequencing data. It takes per-read statistics files as input (generated by [nano_extract](https://github.com/papaKarN/nano_extract)) and produces a single, self-contained, shareable **interactive HTML report** combining tables, histograms, distribution curves, and heatmaps.

## Features

- **Multi-file analysis**: compare several runs/samples in a single report
- **Cumulative statistics table** (Raw / Filtered): number of reads, total bases, mean/median/min/max length, N50, length percentiles, mean/median/percentile quality
- **Summary histograms** (4 panels, horizontal bars)
- **Distribution curves** (length, bases, quality) with 3 display modes: Raw / Filtered / No Outliers
- **Length vs quality heatmaps** (stacked), with the same 3 modes
- **Outlier detection** configurable via a length percentile (`--outlier_percentile`, default 99.5)
- **Quality/length filters** applied live (`--min_len`, `--max_len`, `--min_qual`, `--max_qual`)
- **TXT exports** of statistics (raw and filtered)
- **Minified** HTML report, with an automatically generated **.html.gz** version (opens natively in Firefox/Chrome, ideal for sharing)
- Support for `.gz` files (decompression via `pigz` if available, otherwise `gzip`)
- `--low_memory` mode for very large datasets (sequential processing, memory freed as it goes)
- `--light_html` mode to reduce report size (fewer heatmap bins)

## Installation

```bash
git clone https://github.com/papaKarN/NAVIS.git
cd NAVIS
pip install -r requirements.txt
```

Python 3.8+ recommended. Optional but recommended for large `.gz` files: install [`pigz`](https://zlib.net/pigz/) (parallel decompression).

```bash
# Debian/Ubuntu
sudo apt install pigz
```

## Usage

```bash
python NAVIS.py -i sample1.txt sample2.txt -o report.html
```

### Arguments

| Argument | Description | Default |
|---|---|---|
| `-i`, `--input` | One or more TXT input files (required) | — |
| `-o`, `--output` | Output HTML file | `nanopore_multi_summary.html` |
| `-t`, `--threads` | Number of CPU threads | 0 (use all available CPUs) |
| `-b`, `--bin_size` | Bin size (bp) for heatmaps | 1000 |
| `--min_len` | Minimum read length filter (bp) | none |
| `--max_len` | Maximum read length filter (bp) | none |
| `--min_qual` | Minimum mean quality filter | none |
| `--max_qual` | Maximum mean quality filter | none |
| `--outlier_percentile` | Length percentile above which a read is considered an outlier | 99.5 |
| `--low_memory` | Sequential processing, RAM-efficient | disabled |
| `--light_html` | Reduces heatmap resolution for a lighter file | disabled |

### Full example

```bash
python NAVIS.py \
  -i run1.txt run2.txt run3.txt \
  -o qc_report.html \
  -t 4 \
  -b 500 \
  --min_len 500 --max_len 30000 \
  --min_qual 8 --max_qual 25 \
  --outlier_percentile 99
```

## Input data format

NAVIS expects **TSV files with a header**, such as those produced by [nano_extract](https://github.com/papaKarN/nano_extract):

```
read_id     length    mean_quality
read_001    5432      14.3
read_002    3021      12.8
...
```

- `read_id`: read identifier (not used in calculations, kept for compatibility)
- `length`: read length in bp
- `mean_quality`: mean quality of the read

`.gz` files are supported automatically (`.txt.gz` extension).

## Outputs

For a command with `-o report.html`, NAVIS generates:

- `report.html` — interactive report (minified)
- `report.html.gz` — compressed version of the same report
- raw and filtered statistics in `.txt` format

## Recommended pipeline

```
Nanopore files (FASTQ)
        │
        ▼
  nano_extract  →  .txt files (read_id, length, mean_quality)
        │
        ▼
     NAVIS  →  interactive HTML report
```
